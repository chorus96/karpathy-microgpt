# v2: 영속성 (Persistence)

Karpathy의 [microgpt](https://gist.github.com/karpathy/8627fe009c40f57531cb18360106ce95)는 한 번의 실행에서 학습과 추론을 모두 수행합니다. 스크립트가 종료되면, 학습된 가중치는 사라집니다. 이 버전은 그것을 재사용 가능한 두 단계로 나눕니다: **한 번 학습하고, 여러 번 실행하기**.

## 문제점

원본 `gpt.py`에서는 모든 가중치가 메모리 안에 `Value` 객체로 존재합니다. 직렬화(serialization)가 없습니다. 매 실행마다 단지 20개의 이름을 생성하기 위해 처음부터 몇 분씩 학습에 쓰게 됩니다. 알고리즘을 이해하는 데는 괜찮지만, 모델을 실제로 *사용*하는 데는 적합하지 않습니다.

## 해결책

두 개의 스크립트, 그 사이에 하나의 파일:

```
train.py  ──>  model.json  ──>  run.py
(몇 분)         (저장된 가중치)   (즉시)
```

### `model.json` -- 저장되는 것

출력 파일에는 학습 데이터나 그것을 만들어낸 코드 없이도 모델을 재구성하는 데 필요한 모든 것이 담겨 있습니다:

| 섹션 | 내용 | 필요한 이유 |
|---|---|---|
| `config` | `n_embd`, `n_head`, `n_layer`, `block_size`, `vocab_size` | 동일한 아키텍처를 재구축하기 위해 |
| `chars` | `["<BOS>", "a", "b", ..., "z"]` | 토크나이저를 재구성(텍스트 인코딩/디코딩)하기 위해 |
| `weights` | `{"wte": [[0.01, -0.03, ...], ...], ...}` | 실제로 학습된 지식(약 4,000개의 부동소수점 값) |

이것은 microgpt에서의 `.safetensors`나 `.pt` 체크포인트 파일에 해당합니다. 순수한 JSON입니다 -- 사람이 읽을 수 있고, 이식 가능하며, 불러오는 데 의존성이 없습니다.

### `run.py` -- 추론 시점에 무엇이 다른가

추론 스크립트는 학습과 관련된 모든 것을 건너뜁니다:

- **`Value` 래퍼 없음** -- 가중치는 autograd 노드가 아니라 순수한 Python 부동소수점 값입니다. 계산 그래프도, 기울기 추적도 없습니다. 이는 추론을 눈에 띄게 빠르게 만듭니다.
- **역전파 없음** -- 여기에는 `backward()` 메서드가 존재하지 않습니다. 우리는 순전파만 필요합니다.
- **옵티마이저 없음** -- Adam도, 학습률도, 모멘텀 버퍼도 없습니다.
- **데이터셋 없음** -- 모델은 이미 데이터로부터 학습했습니다. 다시 볼 필요가 없습니다.

순전파(`gpt()`, `linear()`, `softmax()`, `rmsnorm()`)는 학습 버전과 동일하며, 단지 `Value` 객체 대신 순수한 부동소수점 값을 대상으로 작동할 뿐입니다.

## 빠른 시작

**학습** (몇 분 소요, `model.json` 생성):

```bash
./train.sh
```

**실행** (즉시, `model.json`을 불러와 이름 생성):

```bash
./run.sh
```

이것이 두 단계 워크플로입니다. 한 번 학습하고, 원하는 만큼 여러 번 실행하세요.

## CLI 옵션

### train.sh

| 플래그 | 기본값 | 설명 |
|---|---|---|
| `--steps` | `500` | 학습 단계 수 |
| `--output` | `model.json` | 학습된 모델을 저장할 위치 |

```bash
./train.sh --steps 1000 --output my_model.json
```

### run.sh

| 플래그 | 기본값 | 설명 |
|---|---|---|
| `--model` | `model.json` | 저장된 모델의 경로 |
| `--samples` | `20` | 생성할 이름의 개수 |
| `--temperature` | `0.6` | 창의성 제어 (0.1 = 보수적, 1.0 = 대담함) |

```bash
./run.sh --model my_model.json --samples 50 --temperature 0.8
```

## 이것이 가르쳐 주는 것

이것은 모든 실제 ML 시스템이 사용하는 것과 정확히 동일한 패턴을, 단지 뼈대만 남도록 벗겨낸 것입니다:

1. **학습은 산출물(artifact)을 만들어낸다** -- PyTorch에서는 `.pt` 파일이고, llama.cpp에서는 `.gguf`이며, 여기서는 `model.json`입니다. 형식은 중요하지 않습니다. 아이디어는 동일합니다: 학습된 부동소수점 값을 디스크에 직렬화하는 것입니다.

2. **추론에는 autograd가 필요 없다** -- 학습 중에는 기울기가 역방향으로 흐를 수 있도록 모든 연산이 그래프를 구축합니다. 추론 중에는 그저 행렬 연산을 할 뿐입니다. 그래프 오버헤드를 제거하는 것이 바로 프레임워크에 `torch.no_grad()` / `model.eval()` 모드가 있는 이유입니다.

3. **설정(config)은 가중치와 함께 이동해야 한다** -- 가중치는 저장하되 하이퍼파라미터를 잊어버린다면, 모델을 재구성할 수 없습니다. 임베딩 차원이나 헤드의 개수를 알 수 없게 됩니다. 이것이 체크포인트 형식이 항상 원시 숫자와 함께 메타데이터를 묶어 두는 이유입니다.

4. **토크나이저는 가중치와 함께 이동해야 한다** -- 모델은 토큰 ID 5를 글자 "e"와 연관 짓도록 학습했습니다. 다른 토크나이저 매핑으로 가중치를 불러오면, 모든 것이 망가집니다. 토크나이저와 가중치는 분리될 수 없습니다.
