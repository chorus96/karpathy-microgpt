# LLM 용어 정리

이 문서는 이 저장소의 `*.md` 파일들(`README.md`, `model-sources/README.md`, `persistence/README.md`)에서 언급된 LLM(대규모 언어 모델) 관련 용어들의 의미를 자세히 정리한 것입니다. microgpt 코드를 이해하는 데 필요한 개념들을, 실제 대형 모델에서 어떤 의미를 갖는지와 함께 설명합니다.

---

## 1. 모델과 아키텍처 (Model & Architecture)

### GPT (Generative Pre-trained Transformer)
"생성형 사전 학습 트랜스포머". 이전까지의 토큰(문자 또는 단어 조각)들을 보고 **다음 토큰을 예측**하도록 학습하는 언어 모델 계열입니다. 이 예측을 반복하면 텍스트를 한 토큰씩 생성해 낼 수 있습니다. microgpt는 이 알고리즘의 가장 축소된 형태입니다.

### GPT-2
OpenAI가 발표한 GPT 계열의 2세대 모델. microgpt는 GPT-2 스타일 아키텍처를 미니어처로 재현하되, 몇 가지를 단순화했습니다(아래 RMSNorm, ReLU² 참고).

### Transformer (트랜스포머)
GPT의 기반이 되는 신경망 구조. 핵심은 **셀프 어텐션(self-attention)**으로, 시퀀스 내 각 위치가 다른 위치들의 정보를 "참고"하여 문맥을 반영합니다. 여러 개의 트랜스포머 블록(레이어)을 쌓아 만듭니다.

### 레이어 (Layer, `n_layer`)
트랜스포머 블록을 몇 개 쌓을지를 나타내는 값. 각 블록은 어텐션 + MLP로 구성됩니다. microgpt는 `n_layer = 1`로 단 한 개의 블록만 사용합니다. 레이어가 많을수록 표현력이 커지지만 학습이 느려집니다.

### 파라미터 (Parameters, 가중치 / Weights)
모델이 **학습을 통해 조정하는 숫자들**. 행렬(matrix) 형태로 저장되며, 임베딩·어텐션 투영·MLP·출력 헤드 등 모든 연산에 쓰입니다. microgpt는 약 4,000개(`num params: 4064`)의 파라미터를 가집니다. 실제 대형 모델은 수십억~수조 개에 이릅니다. "학습된 지식"은 결국 이 숫자들 안에 담겨 있습니다.

### 하이퍼파라미터 (Hyperparameters)
학습으로 정해지는 파라미터와 달리, 사람이 **미리 설정하는** 구조·학습 설정값. microgpt의 하이퍼파라미터:

| 하이퍼파라미터 | 코드 이름 | microgpt 값 | 의미 |
|---|---|---|---|
| 임베딩 차원 | `n_embd` | 16 | 각 토큰을 표현하는 벡터의 길이 |
| 어텐션 헤드 수 | `n_head` | 4 | 병렬 어텐션의 개수 |
| 레이어 수 | `n_layer` | 1 | 트랜스포머 블록 개수 |
| 컨텍스트 길이 | `block_size` | 8 | 한 번에 볼 수 있는 최대 토큰 수 |
| 어휘 크기 | `vocab_size` | 27 | 토크나이저가 다루는 토큰 종류 수 |

## 2. 토큰화 (Tokenization)

### 토크나이저 (Tokenizer)
텍스트를 모델이 처리할 수 있는 **정수 ID의 나열(토큰)**로 바꾸고(인코딩), 다시 텍스트로 되돌리는(디코딩) 구성 요소. 모델은 문자나 단어가 아니라 이 정수들만 봅니다.

### 문자 단위 토크나이저 (Character-level Tokenizer)
가장 단순한 토크나이저로, **글자 하나하나**에 ID를 부여합니다. microgpt는 알파벳 a–z 각각에 ID를 주어 어휘 크기가 27(문자 26개 + 특수 토큰 1개)이 됩니다.

### 서브워드 / BPE (Subword / Byte Pair Encoding)
실제 대형 모델이 쓰는 방식으로, 단어를 자주 등장하는 **부분 단위(서브워드)**로 쪼갭니다. BPE는 가장 흔한 서브워드 알고리즘입니다. microgpt는 이런 것 없이 순수한 문자만 사용합니다.

### 어휘 크기 (Vocab Size, `vocab_size`)
토크나이저가 구분하는 **토큰 종류의 총 개수**. microgpt는 27. 이 값이 모델 출력층의 크기(로짓 개수)를 결정합니다.

### `<BOS>` (Beginning of Sequence)
"시퀀스 시작"을 나타내는 특수 토큰. microgpt는 이름의 양 끝에 `<BOS>`를 붙여, 어디서 이름이 시작하고 끝나는지를 모델에 알려 줍니다. 생성 시 `<BOS>`가 다시 나오면 이름이 끝난 것으로 간주합니다.

## 3. 자동 미분 (Autograd)

### autograd / 자동 미분 (Automatic Differentiation)
신경망 학습에 필요한 **기울기(gradient)를 자동으로 계산**해 주는 엔진. 각 연산의 결과와, 그 연산이 결과에 미친 영향(미분값)을 추적합니다. PyTorch의 핵심 기능이며, microgpt는 이를 `Value` 클래스로 처음부터 구현합니다.

### `Value` 클래스
microgpt에서 **하나의 스칼라 값**을 감싸는 객체. 값 자체뿐 아니라, 그 값을 만든 연산과 기울기를 함께 저장하여 자동 미분을 가능하게 합니다. Karpathy의 micrograd에서 유래했습니다.

### 순전파 (Forward Pass)
입력에서 출력(그리고 손실)까지 **연산을 앞으로 계산**하는 과정. 각 연산(`+`, `*`, `**`, `exp`, `log`, `relu`)의 결과를 순서대로 구합니다.

### 역전파 (Backward Pass)
출력의 오차를 입력 방향으로 **거꾸로 전파**하며, 각 파라미터가 손실에 얼마나 기여했는지(기울기)를 구하는 과정. `loss.backward()`로 실행됩니다.

### 연쇄 법칙 (Chain Rule)
합성 함수의 미분 규칙. 역전파의 수학적 토대로, 여러 연산이 이어진 경우 각 단계의 미분을 **곱해 나가며** 최종 기울기를 구합니다.

### 계산 그래프 (Computation Graph)
연산들이 서로 어떻게 연결되어 있는지를 나타낸 그래프. 순전파에서 만들어지고, 역전파에서는 이 그래프를 **역위상 순서(reverse topological order)**로 거슬러 올라가며 기울기를 계산합니다.

### 기울기 (Gradient)
각 파라미터를 조금 바꿨을 때 손실이 얼마나 변하는지를 나타내는 값(편미분). 이 방향의 **반대로** 파라미터를 조정하면 손실이 줄어듭니다. 이것이 학습의 원리입니다.

## 4. 모델 구성 요소 (Model Components)

### 임베딩 (Embedding)
토큰 ID(정수)를 **의미를 담은 벡터**로 바꾸는 조회 테이블. microgpt의 임베딩 차원(`n_embd`)은 16입니다.

- **토큰 임베딩 (Token Embedding, `wte`)**: 각 토큰의 정체성을 나타내는 벡터.
- **위치 임베딩 (Position Embedding)**: 토큰이 시퀀스의 **몇 번째 위치**에 있는지를 나타내는 벡터. 트랜스포머는 순서 정보를 내재하지 않으므로 별도로 더해 줍니다.

이 둘을 더해 각 위치의 초기 표현을 만듭니다.

### RMSNorm (Root Mean Square Normalization)
벡터를 안정적인 크기로 **정규화**하는 기법. LayerNorm과 비슷하지만 평균 빼기(shift)와 편향(bias)이 없어 더 단순합니다. microgpt는 LayerNorm 대신 RMSNorm을 사용합니다.

### LayerNorm (Layer Normalization)
일반적인 GPT-2가 사용하는 정규화 기법. 각 벡터를 평균 0, 분산 1로 맞춘 뒤 스케일·이동합니다. microgpt는 이를 더 단순한 RMSNorm으로 대체했습니다.

### 셀프 어텐션 (Self-Attention)
시퀀스 내 각 위치가 다른 위치들을 **얼마나 참고할지**를 스스로 계산하는 메커니즘. 문맥을 반영하는 트랜스포머의 핵심입니다.

### 멀티헤드 어텐션 (Multi-head Attention)
어텐션을 여러 개(`n_head`=4)의 **헤드**로 나누어 병렬 수행하는 방식. 각 헤드가 서로 다른 관계 패턴을 학습할 수 있습니다.

### Q / K / V / O 투영 (Query / Key / Value / Output Projection)
어텐션의 핵심 행렬 연산들:
- **Q (Query, 질의)**: "나는 무엇을 찾고 있는가".
- **K (Key, 키)**: "나는 무엇을 제공하는가".
- **V (Value, 값)**: 실제로 전달되는 정보.
- **O (Output, 출력)**: 헤드들의 결과를 다시 합쳐 내보내는 투영.

각 위치의 Q와 다른 위치들의 K를 비교해 관련도(어텐션 점수)를 구하고, 그 가중치로 V를 합산합니다.

### 내적 어텐션 점수 (Dot-product Attention Scores)
Q와 K의 **내적(dot product)**으로 두 위치의 관련도를 수치화한 값. 값이 클수록 더 관련이 깊다는 뜻이며, 소프트맥스를 거쳐 가중치가 됩니다.

### 소프트맥스 (Softmax)
여러 점수를 **합이 1인 확률 분포**로 바꾸는 함수. 어텐션에서는 점수를 가중치로, 출력층에서는 로짓을 다음 토큰의 확률로 변환하는 데 쓰입니다.

### KV 캐시 (KV Cache)
이미 계산한 위치들의 **K와 V를 저장**해 두어, 토큰을 하나씩 생성할 때 매번 다시 계산하지 않게 하는 효율화 기법. 추론 속도를 크게 높입니다.

### 잔차 연결 (Residual Connection)
레이어의 **출력에 그 레이어의 입력을 그대로 더하는** 연결로, "지름길(skip connection)"이라고도 합니다:

$$\text{출력} = x + F(x)$$

여기서 `x`는 입력, `F(x)`는 레이어가 계산한 변환(어텐션 또는 MLP)입니다.

**microgpt에서의 모습** — 트랜스포머 블록에 잔차 연결이 두 곳(어텐션 뒤, MLP 뒤) 있습니다:

```python
x_residual = x                              # ① 입력을 따로 보관
x = rmsnorm(x)
... 어텐션 또는 MLP 계산 ...
x = [a + b for a, b in zip(x, x_residual)]  # ② 결과 + 원래 입력
```

**왜 필요한가 — 3가지 핵심 이유:**

1. **기울기 소실 방지 (가장 중요)**: 깊은 신경망의 역전파는 연쇄 법칙으로 기울기를 곱해 나가는데, 작은 값이 곱해지면 입력 쪽에서 기울기가 0에 수렴해 학습이 멈춥니다. 잔차가 있으면 미분이 다음과 같습니다:

   $$\frac{\partial (x + F(x))}{\partial x} = 1 + \frac{\partial F(x)}{\partial x}$$

   이 **`1`** 덕분에 기울기가 `F`를 거치지 않고 `x`로 곧장 흐르는 지름길(gradient highway)이 생겨, `F`의 기울기가 아무리 작아도 전체 기울기가 사라지지 않습니다. 코드로는 `Value.__add__`의 역전파(`self.grad += out.grad`; `other.grad += out.grad`)가 바로 이 통로입니다.

2. **항등 함수를 배우기 쉽게**: 잔차가 없으면 "입력을 그대로 통과"시키는 것조차 `F(x)=x`를 학습해야 하지만, 잔차가 있으면 `F(x)=0`만 만들면 `x+0=x`로 통과가 공짜입니다. 즉 각 레이어는 출력 전체가 아니라 **입력에 더할 보정값(잔차, residual)** 만 학습하면 되고, 이것이 이름의 유래입니다.

3. **학습 안정화**: microgpt가 `attn_wo`·`mlp_fc2`를 `std=0`(0에서 시작)으로 초기화하는 것과 맞물립니다. 초기에 `F(x)≈0`이라 블록이 거의 **항등 함수**로 시작해, 안정적으로 변환을 점진 학습합니다.

**비유**: 레이어를 "그림을 수정하는 화가"에 빗대면 — 잔차가 없으면 매번 처음부터 다시 그려 실수가 누적되지만, 잔차가 있으면 원본 위에 **덧칠(보정)만** 해 원본이 항상 보존됩니다.

> 트랜스포머가 수십~수백 층을 안정적으로 쌓을 수 있는 것은 이 잔차 연결 덕분입니다. ResNet(2015)에서 도입되어 지금은 거의 모든 딥러닝 아키텍처의 표준이 되었습니다. 덧셈의 미분은 [`math.md`](math.md)의 미분 법칙을 참고하세요.

### MLP (Multi-Layer Perceptron, 다층 퍼셉트론)
어텐션 뒤에 오는 **피드포워드 신경망**. microgpt에서는 두 개의 선형 레이어와 그 사이의 활성화 함수(ReLU²)로 이루어집니다. 각 위치의 표현을 개별적으로 변환합니다.

### 선형 레이어 (Linear Layer)
입력 벡터에 가중치 행렬을 곱하는 기본 연산(`linear()`). 신경망의 가장 기초적인 변환 단위입니다.

### 활성화 함수 (Activation Function)
선형 연산 사이에 **비선형성**을 넣어 주는 함수. 이것이 없으면 아무리 레이어를 쌓아도 하나의 선형 변환과 같아집니다.
- **ReLU**: 음수를 0으로 만드는 함수(`max(0, x)`).
- **제곱 ReLU / ReLU² (`ReLU(x)^2`)**: ReLU 결과를 제곱한 것. microgpt가 사용합니다.
- **GeLU (Gaussian Error Linear Unit)**: 일반 GPT-2가 쓰는 부드러운 활성화 함수. microgpt는 더 단순한 ReLU²로 대체했습니다.

### 출력 투영 / 출력 헤드 (Output Projection / Output Head)
모델의 마지막 선형 레이어로, 내부 표현을 **어휘 크기(27)만큼의 로짓**으로 변환합니다.

### 로짓 (Logits)
소프트맥스를 적용하기 **전의 원시 점수**들. 각 토큰이 다음에 올 가능성을 나타내며, 소프트맥스를 거쳐 확률이 됩니다.

## 5. 학습 (Training)

### 학습 스텝 (Training Step, `num_steps` / `--steps`)
파라미터를 한 번 업데이트하는 단위. microgpt는 기본 500 스텝을 돌며, 각 스텝은 이름 하나를 처리합니다.

### 손실 함수 / 교차 엔트로피 손실 (Loss / Cross-entropy Loss)
모델의 예측이 정답과 **얼마나 어긋났는지**를 하나의 숫자로 나타낸 값. 교차 엔트로피는 분류 문제(다음 토큰 맞히기)에 표준적으로 쓰이며, 학습은 이 손실을 줄이는 방향으로 진행됩니다. microgpt의 손실은 약 3.3(무작위 추측)에서 시작해 약 2.0까지 떨어집니다.

### 옵티마이저 (Optimizer)
기울기를 이용해 **파라미터를 어떻게 갱신할지** 결정하는 알고리즘.

### Adam (Adaptive Moment Estimation)
가장 널리 쓰이는 옵티마이저. 기울기의 1차·2차 모멘트(이동 평균)를 활용해 파라미터마다 적응적으로 갱신 폭을 조절합니다. microgpt가 사용합니다.

### 모멘텀 버퍼 (Momentum Buffer)
Adam이 과거 기울기의 이동 평균을 저장해 두는 값. 갱신을 부드럽고 안정적으로 만듭니다. 추론 시에는 필요 없습니다.

### 학습률 (Learning Rate)
파라미터를 기울기 방향으로 **얼마나 크게** 움직일지 정하는 값. 너무 크면 불안정하고, 너무 작으면 학습이 느립니다.

### 학습률 감소 (Learning Rate Decay)
학습이 진행됨에 따라 학습률을 **점점 줄이는** 전략. microgpt는 선형(linear) 감소를 사용해, 후반부에 더 세밀하게 수렴하도록 합니다.

### 가우시안 초기화 (Gaussian Initialization)
학습을 시작하기 전, 가중치를 **작은 무작위 정규분포(가우시안) 값**으로 채우는 것. 대칭성을 깨고 안정적인 학습을 돕습니다.

---

## 6. 추론과 생성 (Inference & Generation)

### 추론 (Inference)
학습이 끝난 모델을 **실제로 사용해 결과를 만들어 내는** 단계. microgpt에서는 새 이름을 생성하는 과정입니다. 학습과 달리 역전파·옵티마이저·데이터셋이 필요 없어 더 빠릅니다.

### 확률 분포 (Probability Distribution)
소프트맥스를 거친 로짓으로, **각 토큰이 다음에 올 확률**을 모아 놓은 것(합이 1). 여기서 다음 토큰을 뽑습니다.

### 샘플링 (Sampling)
확률 분포에서 **다음 토큰을 무작위로 선택**하는 것. 항상 최고 확률만 고르지 않기 때문에 다양한 결과가 나옵니다.

### 온도 (Temperature, `--temperature`)
샘플링의 **무작위성(창의성)을 조절**하는 값. 낮으면(예: 0.1) 보수적이고 예측 가능하며, 높으면(예: 1.0) 대담하고 다양해집니다. microgpt 기본값은 0.6입니다.

---

## 7. 영속성과 배포 (Persistence & Deployment)

### 직렬화 (Serialization)
메모리 속 모델(가중치 등)을 **파일로 저장 가능한 형태**로 바꾸는 것. 원본 microgpt에는 없으며, `persistence/` 버전이 이를 추가합니다.

### 체크포인트 (Checkpoint)
학습된 모델의 상태(가중치·설정·토크나이저)를 저장한 파일. "한 번 학습해 저장하고, 여러 번 불러 쓴다"는 실제 ML 워크플로의 핵심입니다.
- **`.pt`**: PyTorch 체크포인트 형식.
- **`.safetensors`**: 안전한 텐서 저장 형식.
- **`.gguf`**: llama.cpp가 쓰는 형식.
- **`model.json`**: microgpt의 체크포인트. 순수 JSON으로, 설정·문자 목록·가중치를 담습니다.

### 설정 (Config)
모델을 재구성하는 데 필요한 하이퍼파라미터 묶음(`n_embd`, `n_head`, `n_layer`, `block_size`, `vocab_size`). 가중치만 있고 설정이 없으면 모델을 복원할 수 없으므로, 체크포인트에는 항상 설정이 함께 저장됩니다.

### `torch.no_grad()` / `model.eval()`
PyTorch에서 **추론 모드**로 전환하는 방법. `no_grad()`는 기울기 추적(계산 그래프 구축)을 끄고, `eval()`은 모델을 평가 모드로 바꿉니다. 그래프 오버헤드를 없애 추론을 빠르게 만듭니다. microgpt 추론 버전이 애초에 `Value` 래퍼를 쓰지 않는 것과 같은 원리입니다.
