# microgpt

**243줄의 순수한, 의존성 없는 Python**으로 학습하고 실행하는 GPT입니다. PyTorch도, NumPy도, 그 무엇도 없습니다. 오직 `import math`와 순수한 의지뿐입니다.

[Andrej Karpathy](https://x.com/karpathy)의 [microgpt](https://gist.github.com/karpathy/8627fe009c40f57531cb18360106ce95)를 기반으로 합니다 -- 존재할 수 있는 가장 순수하게 증류된 형태의 GPT 알고리즘입니다.

> *"이것이 필요한 알고리즘의 전부입니다. 나머지는 모두 효율성을 위한 것일 뿐입니다. 저는 이것을 더 이상 단순화할 수 없습니다."*
> -- Andrej Karpathy

## Karpathy는 누구인가요?

Andrej Karpathy는 현대 AI에서 가장 영향력 있는 인물 중 한 명입니다. 그는 OpenAI의 창립 멤버였고, Tesla의 오토파일럿 비전 팀을 이끌었으며, 딥러닝을 이해하기 쉽게 만드는 비범한 재능을 지니고 있습니다. 그의 [YouTube 강의](https://www.youtube.com/@AndrejKarpathy), 블로그 글, 그리고 [nanoGPT](https://github.com/karpathy/nanoGPT)와 [micrograd](https://github.com/karpathy/micrograd) 같은 프로젝트는 대부분의 대학 강의를 합친 것보다 더 많은 사람들에게 신경망이 실제로 어떻게 작동하는지 가르쳐 주었습니다.

## 빠른 시작

```bash
./train.sh
```

이게 전부입니다. 이 스크립트는 필요하면 [uv](https://docs.astral.sh/uv/)를 설치한 후 `gpt.py`를 실행합니다. 최신 기기에서 몇 분 정도 걸립니다 -- GPU는 필요 없습니다.

## 실제로 무엇을 하나요?

전체 스크립트를 단계별로 살펴보겠습니다. 위에서 아래로 읽으세요 -- 그것이 실행되는 순서입니다.

### 1. 데이터셋 불러오기

[makemore](https://github.com/karpathy/makemore)에서 약 32,000개의 사람 이름(예: "emma", "olivia", "ava")을 다운로드합니다. 이것들이 모델이 학습하는 "문서"입니다. 여전히 알아볼 수 있는 결과물을 만들어내는 가장 작은 텍스트 말뭉치라고 생각하면 됩니다.

### 2. 토크나이저 만들기

문자 단위 토크나이저를 만듭니다: 각 고유 문자(a-z)가 정수 ID를 받고, 여기에 특수 토큰 `<BOS>`(Beginning of Sequence, 시퀀스 시작)가 더해집니다. 어휘 크기는 결국 27이 됩니다. 이것이 모델이 보는 알파벳입니다 -- 서브워드도, BPE도 없이 오직 순수한 문자뿐입니다.

### 3. autograd를 처음부터 구현하기

`Value` 클래스는 아주 작은 자동 미분 엔진입니다(Karpathy의 [micrograd](https://github.com/karpathy/micrograd)의 후예입니다). 모든 스칼라는 다음을 추적하는 `Value`로 감싸집니다:

- **순전파(forward pass)**: 각 연산(`+`, `*`, `**`, `exp`, `log`, `relu`)의 결과를 계산합니다
- **역전파(backward pass)**: 계산 그래프를 역위상 순서로 거슬러 올라가며 연쇄 법칙(chain rule)을 통해 기울기(gradient)를 계산합니다

이것은 PyTorch의 `autograd`가 기반하고 있는 것과 동일한 아이디어입니다 -- 단지 C++ 백엔드, CUDA 커널, 그리고 200만 줄의 코드가 없을 뿐입니다.

### 4. 모델 파라미터 초기화하기

GPT-2 스타일의 아키텍처를 축소판으로 구성합니다:

| 하이퍼파라미터 | 값 |
|---|---|
| 임베딩 차원 | 16 |
| 어텐션 헤드 | 4 |
| 레이어 | 1 |
| 컨텍스트 길이 | 8 |
| **총 파라미터** | **~4,000** |

가중치 행렬(토큰 임베딩, 위치 임베딩, 어텐션 Q/K/V/O 투영, MLP 레이어, 출력 헤드)은 작은 무작위 가우시안 값으로 초기화됩니다. 모든 개별 가중치는 autograd 그래프의 `Value` 노드입니다.

### 5. 순전파 정의하기

`gpt()` 함수는 GPT-2 변형의 한 토큰 단계를 구현합니다:

1. **임베딩(Embed)** -- 토큰 임베딩과 위치 임베딩을 조회하여 함께 더합니다
2. **RMSNorm** -- 벡터를 정규화합니다(LayerNorm과 비슷하지만 더 단순하며, 편향/이동이 없습니다)
3. **멀티헤드 셀프 어텐션(Multi-head self-attention)** -- 4개의 헤드 각각이 Q/K/V 투영, 내적 어텐션 점수, 소프트맥스 가중치, 그리고 값들의 가중 합을 계산합니다. 위치 간 효율성을 위해 KV 캐시를 사용합니다
4. **잔차 연결(Residual connection)** -- 어텐션 출력을 다시 입력에 더합니다
5. **MLP** -- 그 사이에 제곱 ReLU 활성화 함수(`ReLU(x)^2`)가 있는 두 개의 선형 레이어입니다
6. **또 다른 잔차 연결** -- MLP 출력을 다시 더합니다
7. **출력 투영(Output projection)** -- 어휘 크기의 로짓(logits)으로 매핑하는 선형 레이어입니다

일반적인 GPT-2와의 주요 차이점: LayerNorm 대신 RMSNorm, 편향이 전혀 없음, GeLU 대신 ReLU^2.

### 6. Adam으로 학습하기

500단계 동안, 학습 루프는 다음을 수행합니다:

1. 데이터셋에서 **이름 하나를 선택**합니다
2. 양 끝에 `<BOS>` 구분자를 붙여 **토큰화**합니다
3. 토큰별로 **순전파를 실행**하여 교차 엔트로피 손실(cross-entropy loss)까지 이르는 계산 그래프를 구축합니다
4. **역전파를 실행**(`loss.backward()`)하여 역방향 자동 미분을 통해 약 4,000개 파라미터 전체의 기울기를 계산합니다
5. 선형 학습률 감소를 적용한 Adam 옵티마이저로 **가중치를 업데이트**합니다

각 단계는 하나의 이름을 처리합니다. 손실은 약 3.3(27개 문자에 대한 무작위 추측)에서 시작하여 약 2.0까지 떨어집니다.

### 7. 이름 생성하기

학습이 끝나면, 모델은 다음 방식으로 20개의 새로운 이름을 생성합니다:

1. `<BOS>` 토큰으로 시작합니다
2. 순전파를 실행하여 다음 문자에 대한 확률 분포를 얻습니다
3. 그 분포에서 샘플링합니다(제어된 무작위성을 위해 `temperature=0.6` 사용)
4. `<BOS>`가 다시 예측되거나(이름의 끝) 최대 길이에 도달할 때까지 반복합니다

`lellen`, `aman`, `lela`, `kan` 같은 출력을 보게 될 것입니다 -- 실제 이름은 아니지만, 그럴듯해 *보입니다*.

## 다음에 시도해 볼 것들

- **`num_steps`를 500 이상으로 늘려 보세요** -- 손실이 떨어지고 이름이 더 사실적으로 변하는 것을 지켜보세요
- **추론 시 `temperature`를 조정해 보세요** -- 낮을수록 보수적이고, 높을수록 창의적입니다
- **레이어를 추가**(`n_layer = 2`)하거나 임베딩 크기를 늘려 보세요(`n_embd = 32`) -- 파라미터가 많아지고, 모델은 좋아지지만, 학습은 느려집니다
- **데이터셋을 교체해 보세요** -- `input.txt`를 줄 단위로 구분된 어떤 텍스트로든 바꿔 보세요(도시 이름, 포켓몬, 어떤 언어의 단어든)
- **속도를 위해 PyPy를 사용하세요** -- `uv run --python pypy gpt.py`는 더 나은 JIT와 가비지 컬렉션 덕분에 상당한 속도 향상을 제공합니다([출처](https://gist.github.com/karpathy/8627fe009c40f57531cb18360106ce95?permalink_comment_id=5427633#gistcomment-5427633))
- **코드를 읽어 보세요** -- 243줄이며, 완전히 주석이 달려 있고, 진정으로 아름답습니다. GPT가 근본적인 수준에서 어떻게 작동하는지 이해하는 데 이보다 더 좋은 방법은 없습니다
